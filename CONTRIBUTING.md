# Contributing to semrel-registry

Thanks for contributing to the semrel registry.

## Scope

This repository contains the public plugin index, registry documentation, schemas, and automation that keeps the index in sync.

## Ways to contribute

- submit or update plugin metadata
- improve registry and release documentation
- enhance schema validation or CI workflows
- report issues with `plugins.json` or publishing

## Contribution workflow

1. Fork the repository or create a branch from `main`.
2. Keep changes focused and explain the motivation in the pull request.
3. Update docs when changing contributor-facing behavior.
4. Validate any changed JSON or workflow files before opening a PR.
5. Request review once the change is ready to merge.

## Pull request expectations

- describe what changed and why
- link related issues or plugin repositories when relevant
- include examples for API or schema changes
- keep unrelated formatting changes out of the PR

## Local validation

This repository starts with documentation and static files only. Before opening a PR, at minimum:

- confirm Markdown links resolve
- confirm JSON files are valid
- confirm workflow YAML stays syntactically valid

## Code of conduct

Be respectful, constructive, and collaborative in all discussions and reviews.
