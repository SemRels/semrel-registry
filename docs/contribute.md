# Plugin submission overview

This document explains how plugins are added to the go-semrel registry.

## Submission flow

1. Publish the plugin as a GitHub Release in `SemRels/go-semrel-plugins`.
2. Follow the release tag, binary naming, and checksum conventions from the release guide.
3. Registry automation validates the discovered release against the registry schema.
4. Valid releases are aggregated into `plugins.json` and published automatically.
5. Open a pull request to this repository only when documentation, schemas, or automation need to change.

## What maintainers will review

- release naming and asset structure
- version and compatibility information
- checksum completeness and consistency
- schema and automation changes proposed in pull requests

## Notes

- `plugin-metadata.json` defines the registry payload expected by the automation.
- `plugins.json` is generated by automation and should not be hand-edited except for bootstrapping or maintenance work.
- The public registry will ultimately be available under `registry.semrel.io`.
