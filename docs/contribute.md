# Plugin submission overview

This document explains how plugins are added to the go-semrel registry.

## Submission flow

1. Publish the plugin from its own GitHub repository.
2. Prepare the plugin metadata expected by this registry.
3. Open a pull request against this repository.
4. Wait for schema validation and maintainer review.
5. After merge, the registry index is regenerated and published.

## What maintainers will review

- plugin identity and repository URL
- release version and compatibility information
- metadata completeness
- consistency with the registry schema

## Notes

- The concrete `plugin-metadata.json` schema will be added separately.
- `plugins.json` is generated by automation and should not be hand-edited except for bootstrapping or maintenance work.
- The public registry will ultimately be available under `registry.semrel.io`.
