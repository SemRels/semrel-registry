# Registry API reference

The registry exposes a static JSON index from the repository root.

## Endpoint

- `GET /plugins.json`

This file is designed to be reachable from GitHub Pages and later from `https://registry.semrel.io/plugins.json`.

## Current payload shape

```json
{
  "schemaVersion": 1,
  "generatedAt": null,
  "plugins": []
}
```

## Fields

- `schemaVersion` - version of the registry payload format
- `generatedAt` - timestamp written by automation when the index is generated
- `plugins` - array of plugin records collected from validated metadata

## Response characteristics

- content type: `application/json`
- source of truth: repository root file committed or generated by CI
- hosting: GitHub Pages from the `main` branch root
- client update strategy: semrel clients compare `plugins[].versions` and select the newest stable (`prerelease: false`) version for update checks

## Schema reference

The registry payload is validated against the semrel plugin metadata schema. Author-facing publishing guidance lives in the [release guide](release-guide.md).
