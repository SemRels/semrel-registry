Submit a Plugin

Fill in the form — it opens a pre-filled GitHub Issue for maintainer review. Your plugin must be based on the plugin-template.

Must be based on SemRels/plugin-template
Format: category-name, e.g. updater-pypi

Plugin standards

○ Naming convention: category-name
○ SECURITY.md present
○ CONTRIBUTING.md present
○ GOVERNANCE.md present
○ LICENSE file present (Apache-2.0)
○ .github/workflows/release.yml with multi-arch build
○ .github/workflows/security.yml with REUSE + govulncheck
○ go.mod present (Go plugin)
○ cmd/plugin/ entry point
○ release.yml triggers registry sync on new tag

Review process

1
Validate
Click "Validate →" to check your repo against SemRels standards.
2
Submit
A pre-filled GitHub Issue is opened for maintainer review.
3
Merge
Approved plugins are added to plugins.json via pull request.
4
Auto-sync
New releases are auto-fetched from GitHub Releases into the registry.
